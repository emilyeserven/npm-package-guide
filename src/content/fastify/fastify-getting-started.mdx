<SectionIntro>
Let's set up a minimal Fastify server. Fastify v5 is the current major, and it fully supports ESM out of the box.
</SectionIntro>

<SectionSubheading id="toc-minimal-server">Minimal Server</SectionSubheading>

<CodeAccordion title="server.ts" language="TypeScript" startOpen>
{`import Fastify from 'fastify'

const app = Fastify({
  logger: true,  // Built-in pino logger
})

// Declare a route
app.get('/', async (request, reply) => {
  return { hello: 'world' }
})

// Start the server
try {
  await app.listen({ port: 3001 })
} catch (err) {
  app.log.error(err)
  process.exit(1)
}`}
</CodeAccordion>

That's it. Fastify returns JSON by default when you return an object from an async handler. No `res.json()` needed. The logger outputs structured JSON via pino, which is orders of magnitude faster than `console.log`.

Using `tsx` gives you hot-reload in development without a full build step. For production, compile with `tsc` and run the plain JS output.

<SectionSubheading id="toc-key-differences">Key Differences from Express Handlers</SectionSubheading>

<SectionList>
  <ColItem>**Return values** â€” In async handlers, `return { data }` sends JSON automatically. No need for `res.json()`.</ColItem>
  <ColItem>**Request/Reply** â€” Fastify uses `request` and `reply` (not `req`/`res`). The `reply` object has methods like `reply.code(201)` and `reply.header()`.</ColItem>
  <ColItem>**Built-in logger** â€” Every request gets a child logger with a unique request ID. Use `request.log.info()` instead of `console.log()`.</ColItem>
  <ColItem>**No callback style** â€” Fastify is async-first. All handlers and hooks should be async functions.</ColItem>
</SectionList>
