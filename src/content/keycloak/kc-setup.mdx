<SectionSubheading id="toc-docker">Local Development with Docker</SectionSubheading>

The fastest way to get Keycloak running for local development:

<CodeAccordion title="docker-compose.yml" language="yaml" startOpen>
{`# docker-compose.yml
services:
  keycloak:
    image: quay.io/keycloak/keycloak:26.4
    command: start-dev
    environment:
      KC_BOOTSTRAP_ADMIN_USERNAME: admin
      KC_BOOTSTRAP_ADMIN_PASSWORD: admin
      KC_DB: postgres
      KC_DB_URL: jdbc:postgresql://db:5432/keycloak
      KC_DB_USERNAME: keycloak
      KC_DB_PASSWORD: keycloak
    ports:
      - "8080:8080"
    depends_on:
      - db
  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: keycloak
      POSTGRES_USER: keycloak
      POSTGRES_PASSWORD: keycloak
    volumes:
      - pgdata:/var/lib/postgresql/data
volumes:
  pgdata:`}
</CodeAccordion>

<Gotcha>
The `start-dev` command disables HTTPS and caching — never use this in production. Dev mode is for local experimentation only.
</Gotcha>

<SectionSubheading id="toc-realm">Creating Your Realm</SectionSubheading>

<SectionList>
  <ColItem>**1. Access the Admin Console** — Navigate to `http://localhost:8080` and log in with the bootstrap admin credentials.</ColItem>
  <ColItem>**2. Create a new Realm** — Click the realm dropdown (top-left, shows "master") and select **Create Realm**. Name it something like `my-app`. Never put application users in the master realm.</ColItem>
  <ColItem>**3. Create a test User** — Go to **Users > Add user**. Set a username, then go to the **Credentials** tab to set a password. Toggle off "Temporary" if you don't want a forced password change on first login.</ColItem>
  <ColItem>**4. Configure Roles** — Under **Realm Roles**, create roles like `admin`, `editor`, `viewer`. Assign them to your test user via **Users > Role Mapping**.</ColItem>
</SectionList>

<SectionSubheading id="toc-client">Registering Your React SPA</SectionSubheading>

<SectionList>
  <ColItem>**1. Create the Client** — Go to **Clients > Create client**. Set Client ID to `react-app`. Choose **OpenID Connect** as the protocol.</ColItem>
  <ColItem>**2. Configure Access** — Set **Client authentication** to **OFF** (public client). Enable **Standard flow** (Authorization Code). Disable Direct access grants.</ColItem>
  <ColItem>**3. Set Redirect URIs** — Add `http://localhost:5173/*` for Vite dev. Set **Web Origins** to `http://localhost:5173` for CORS.</ColItem>
  <ColItem>**4. Discovery URL** — Your OIDC authority: `http://localhost:8080/realms/my-app`. Full discovery doc at `http://localhost:8080/realms/my-app/.well-known/openid-configuration`.</ColItem>
</SectionList>
