<SectionSubheading id="toc-cli">CLI Usage</SectionSubheading>

```bash
# Basic usage
npx jscodeshift -t ./transforms/rename-api.ts src/

# Dry run (don't write files)
npx jscodeshift -t ./transforms/rename-api.ts -d src/

# Print changes to stdout
npx jscodeshift -t ./transforms/rename-api.ts -p src/

# Specify parser, workers, extensions
npx jscodeshift \
  -t ./transforms/rename-api.ts \
  --parser=tsx \
  --cpus=4 \
  --extensions=ts,tsx \
  --ignore-pattern="**/node_modules/**" \
  src/

# Pass custom options to your transform
npx jscodeshift -t ./transforms/rename-api.ts \
  --oldName=fetch --newName=httpGet \
  src/
```

<SectionSubheading id="toc-versioning">Versioning & Organizing Codemods</SectionSubheading>

<SectionNote>
Keep codemods in your repo alongside the code they modify. Name them with a date or version prefix for chronological ordering: `2025-01-rename-api.ts`, `2025-02-migrate-query-v5.ts`. Archive after running — don't delete, they document migration history.
</SectionNote>
