<SectionSubheading id="toc-concepts">{'\u{1F4D6}'} Key terms</SectionSubheading>

<DefinitionTable termHeader="Term" descHeader="Definition">
  <DefRow term="Chart">A package of templated K8s YAML files. Like an npm package but for infrastructure. Has a name, version, and dependencies.</DefRow>
  <DefRow term="values.yaml">The config file you customize &mdash; like .env but for your whole deployment. Sets replicas, image versions, resource limits, feature flags.</DefRow>
  <DefRow term="Templates">K8s YAML files with <code>{"{{ .Values.x }}"}</code> placeholders. Helm fills them in from values.yaml. Like JSX but for infrastructure.</DefRow>
  <DefRow term="Release">A deployed instance of a chart. You can have multiple releases of the same chart (dev, staging, prod) with different values.</DefRow>
  <DefRow term="helm install">Deploys a chart to your cluster &mdash; like &ldquo;npm install&rdquo; + &ldquo;npm start&rdquo; combined.</DefRow>
  <DefRow term="helm upgrade">Updates a running release with new values or chart version. Like deploying a new version of your app.</DefRow>
</DefinitionTable>

<SectionSubheading id="toc-takeaway">{'\u{1F4A1}'} Key takeaway</SectionSubheading>

<Explainer title="Remember this">
Helm = templated, reusable, versionable K8s deployments. Charts bundle everything needed to deploy an app. <code>values.yaml</code> is your control panel.
</Explainer>
