<SectionSubheading id="toc-gotchas">{'\u26A0\uFE0F'} Common error states</SectionSubheading>

<SectionList>
<ColItem>**ImagePullBackOff** &mdash; K8s can't pull the container image. Usually a typo in the image name, missing tag, or authentication issue with a private registry.</ColItem>
<ColItem>**CrashLoopBackOff** &mdash; Container starts but keeps crashing. Check logs with <code>kubectl logs &lt;pod&gt; --previous</code> to see the error from the last crash.</ColItem>
<ColItem>**OOMKilled** &mdash; Container exceeded its memory limit. Increase the memory limit in your resource spec or optimize your app's memory usage.</ColItem>
<ColItem>**Pending** &mdash; Pod can't be scheduled. Usually means insufficient cluster resources (CPU/memory), or the node selector/affinity rules can't be satisfied.</ColItem>
</SectionList>

<SectionSubheading id="toc-takeaway">{'\u{1F4A1}'} Key takeaway</SectionSubheading>

<Explainer title="Remember this">
<code>kubectl</code> is your DevTools for K8s. Master <code>get</code>, <code>describe</code>, <code>logs</code>, and <code>exec</code> &mdash; they'll help you diagnose most issues. Check events when pods won't start.
</Explainer>
