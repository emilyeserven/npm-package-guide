<SectionIntro>
The foundation everything runs on. Containers solve the same problem as <code>node_modules</code> + <code>.nvmrc</code> + <code>package-lock.json</code>, but for an entire application environment.
</SectionIntro>

<SectionSubheading id="toc-frontend">{'\u{1F5A5}\uFE0F'} Your frontend context</SectionSubheading>

<SectionNote>
You know how <code>node_modules</code> gives you a reproducible dependency tree? Containers do that for an entire application environment.
</SectionNote>

<SectionSubheading id="toc-infra">{'\u2601\uFE0F'} Infrastructure reality</SectionSubheading>

<SectionList>
<ColItem>A container packages your app code + its runtime + dependencies + OS libraries into one portable unit. Docker is the tool that builds and runs containers.</ColItem>
</SectionList>
