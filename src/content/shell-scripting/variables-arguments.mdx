<SectionSubheading id="toc-assignment">Variable Assignment</SectionSubheading>

No spaces around `=`. This is the #1 gotcha for beginners.

<CodeAccordion title="variables.sh" ext="bash">
```bash
#!/usr/bin/env bash

# ‚úÖ Correct ‚Äî no spaces around =
NAME="world"
COUNT=42

# ‚ùå WRONG ‚Äî bash thinks "NAME" is a command
# NAME = "world"

# Reference with $
echo "Hello, $NAME!"

# Curly braces for clarity
PREFIX="deploy"
echo "${PREFIX}_staging"  # ‚Üí deploy_staging
```
</CodeAccordion>

<SectionSubheading id="toc-quoting">Quoting Rules</SectionSubheading>

Double quotes allow variable expansion; single quotes treat everything literally.

<CodeAccordion title="Quoting" ext="bash">
```bash
DIR="my project"

# Double quotes: variables expand, spaces preserved
echo "Dir is: $DIR"   # ‚Üí Dir is: my project

# Single quotes: everything is literal
echo 'Dir is: $DIR'   # ‚Üí Dir is: $DIR

# ‚ö†Ô∏è ALWAYS quote variables to handle spaces in paths
ls "$DIR"    # ‚úÖ One arg: "my project"
ls $DIR     # ‚ùå Two args: "my" and "project"
```
</CodeAccordion>

<SectionSubheading id="toc-args">Script Arguments & Command Substitution</SectionSubheading>

<CodeAccordion title="args.sh" ext="bash">
```bash
#!/usr/bin/env bash

echo "Script: $0"       # Script name
echo "First:  $1"       # First argument
echo "All:    $@"       # All arguments
echo "Count:  $#"       # Number of arguments

# Command substitution: capture output into a variable
BRANCH=$(git branch --show-current 2>/dev/null)
FILES=$(ls -1 | wc -l)
echo "On $BRANCH with $FILES files"
```
</CodeAccordion>
