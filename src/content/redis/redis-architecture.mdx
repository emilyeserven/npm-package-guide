<SectionIntro>
Understanding how Redis works under the hood helps you make better decisions about when to use it and how to configure it. Here are the architectural concepts that matter most.
</SectionIntro>

<SectionSubheading id="toc-overview">Overview</SectionSubheading>

Redis achieves its speed through three architectural choices: keeping all data in memory, processing commands on a single thread (no locking overhead), and using efficient data structure implementations optimized for each type.
