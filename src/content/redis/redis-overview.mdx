<SectionSubheading id="toc-what-is">What is Redis?</SectionSubheading>

Think of Redis as `localStorage` for your server. Just like `localStorage` lets your frontend store data in the browser for fast access, Redis lets your API server store data in RAM. The difference? Redis is shared across all users, supports rich data structures, and auto-expires data with TTL.

When your dashboard calls `fetch('/api/user/me')`, the API checks Redis first. If the data is there (a "cache hit"), it returns in under 1ms. If not (a "cache miss"), it queries the database (~50ms), caches the result in Redis for next time, and returns it.

But caching is just the beginning. Redis also powers session management (how your login persists), rate limiting (how your API blocks abuse), real-time features (how your chat messages arrive instantly), and more. It's the invisible layer behind most of the fast apps you use every day.
