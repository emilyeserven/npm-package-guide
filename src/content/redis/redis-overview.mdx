<SectionSubheading id="toc-what-is">What is Redis?</SectionSubheading>

Think of Redis as the layer that sits between your application and your database. Instead of hitting PostgreSQL every time someone loads a page, you check Redis first. If the data is there (a "cache hit"), you return it instantly. If not (a "cache miss"), you query the database, store the result in Redis for next time, and return it.

But caching is just the beginning. Redis is also used for session management, rate limiting, real-time leaderboards, pub/sub messaging, job queues, and more. It's one of those tools that shows up in virtually every production stack.
