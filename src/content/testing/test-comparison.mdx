<SectionSubheading id="toc-when">{'\uD83E\uDD14'} When to use each type</SectionSubheading>

<SectionList>
<ColItem>**Use unit tests** when you're testing pure logic &mdash; functions that take inputs and return outputs without needing a DOM or browser. This is the cheapest place to catch bugs.</ColItem>
<ColItem>**Use component tests** when you need to verify that a UI element renders correctly and responds to user interaction. These catch rendering bugs and interaction issues that unit tests can't.</ColItem>
<ColItem>**Use E2E tests** when you need to verify a complete user workflow that spans multiple pages, involves authentication, or requires real API calls. Reserve these for your most critical paths.</ColItem>
</SectionList>
