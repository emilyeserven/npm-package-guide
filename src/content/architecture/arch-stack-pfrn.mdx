<SectionIntro>
The **PFRN** stack is a modified version of MERN that swaps MongoDB for **PostgreSQL** (a more structured database) and Express for **Fastify** (a faster server framework), while keeping React and Node.js the same. This demonstrates a key architecture principle: **you can swap individual layers without rewriting everything else**.
</SectionIntro>

<SectionSubheading id="toc-overview">{'\u{1F4CB}'} Overview</SectionSubheading>

<SectionList>
<ColItem>**Stronger data layer** &mdash; PostgreSQL<FnRef n={1} /> enforces data integrity through schemas, constraints, and relationships. Much harder to save bad data compared to MongoDB.</ColItem>
<ColItem>**Faster server** &mdash; Fastify handles significantly more requests per second than Express<FnRef n={3} />, with built-in schema validation and a cleaner plugin architecture.</ColItem>
<ColItem>**Same JavaScript ecosystem** &mdash; React and Node.js stay the same, so you keep the one-language benefit of MERN while upgrading the backend.</ColItem>
</SectionList>

<Explainer title="PFRN &mdash; a backend analogy">
Think of PFRN as upgrading your backend service from a prototype to production. It's like swapping a Python dictionary-based cache for Redis, or replacing a simple Flask server with FastAPI — you get better performance, stronger data validation, and more structure, but you need more upfront design. The MERN-to-PFRN shift mirrors how backend teams typically evolve: start with something flexible, then tighten things up once you understand your data model.
</Explainer>
