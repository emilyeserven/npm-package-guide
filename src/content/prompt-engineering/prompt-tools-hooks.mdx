<SectionIntro>
Hooks<FnRef n={1} /> are user-defined shell commands that run automatically at specific points in the Claude Code lifecycle &mdash; before a tool runs, after a tool runs, or when a notification fires. They let you enforce project rules, auto-format code, block dangerous operations, and integrate with external services without any manual intervention.
</SectionIntro>

<SectionSubheading id="toc-overview">How Hooks Work</SectionSubheading>

Hooks are configured in <code>.claude/settings.json</code> (project-level, shared via git) or <code>~/.claude/settings.json</code> (global, personal). Each hook specifies a matcher (which tool triggers it) and a shell command to run.

Four hook types cover the full lifecycle:

<SectionList>
<ColItem><strong>PreToolUse</strong> &mdash; Runs before a tool executes. Can block the operation by exiting with a non-zero status code. Use for safety guards and validation.</ColItem>
<ColItem><strong>PostToolUse</strong> &mdash; Runs after a tool completes. Use for auto-formatting, linting, running tests, or logging.</ColItem>
<ColItem><strong>Notification</strong> &mdash; Runs when Claude sends a notification (e.g., task complete). Use for desktop alerts, Slack messages, or email.</ColItem>
<ColItem><strong>Stop</strong> &mdash; Runs when Claude finishes a turn. Use for session logging or cleanup.</ColItem>
</SectionList>
