<SectionIntro>
Slash commands<FnRef n={1} /> (also called &ldquo;skills&rdquo;) are reusable prompt templates that you invoke with <code>/project:name</code> or <code>/user:name</code>. Define them as Markdown files, share them via git, and standardize AI workflows across your team.
</SectionIntro>

<SectionSubheading id="toc-overview">How Slash Commands Work</SectionSubheading>

Each command is a Markdown file whose entire content becomes the prompt. The <code>$ARGUMENTS</code> placeholder gets replaced with whatever the user types after the command name, making commands flexible enough for many use cases.

<ToolDetail toolId="skills" section="overview" />

<SectionSubheading id="toc-best-for">Best Use Cases</SectionSubheading>

Slash commands are most valuable for tasks you repeat frequently or need to be consistent across a team. They turn multi-paragraph prompts into a single command.

<ToolDetail toolId="skills" section="bestFor" />

<SectionSubheading id="toc-creating">Creating Commands</SectionSubheading>

Setting up slash commands takes just a few minutes. Project-level commands live in <code>.claude/commands/</code> and are shared via version control; user-level commands live in <code>~/.claude/commands/</code> and are personal.
