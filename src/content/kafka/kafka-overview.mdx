<SectionIntro>
Apache Kafka is a **distributed event streaming platform**. Think of it as a massive, fault-tolerant, ordered log of events that multiple applications can write to and read from simultaneously. Originally built at LinkedIn to handle their enormous data pipeline, it now processes trillions of events per day at companies worldwide.
</SectionIntro>

<SectionSubheading id="toc-what-is">What Makes Kafka Different?</SectionSubheading>

Unlike a traditional message queue where messages are deleted after consumption, Kafka **retains events for a configurable period** (or forever). This means multiple consumers can read the same data independently, replay events, and build entirely different views of the same stream.

<Explainer>
Think of Kafka like a newspaper: reporters (producers) write articles, Kafka publishes and *keeps* them, and any number of readers (consumers) can go back and read past editions at any time. Traditional message queues are more like a phone call â€” once the message is delivered, it's gone.
</Explainer>
